# 系统设计说明

## 总体结构

系统采用 Streamlit 多页面结构。首页位于 `app/main.py`，功能页面位于 `app/pages/`，公共逻辑位于 `app/utils/`，数据文件位于 `data/`，测试文件位于 `tests/`。

## 页面层

- `app/main.py`：首页驾驶舱，展示软件定位、数据资产和模块入口。
- `app/pages/1_实验导学.py`：课程导学和模型说明。
- `app/pages/2_文献数据导入.py`：文献 CSV 上传、字段校验和 validated 数据保存。
- `app/pages/3_文献数据库.py`：数据筛选、样本详情和图表展示。
- `app/pages/4_虚拟实验.py`：参数输入、结果计算和场景对比。
- `app/pages/5_二维交互实验台.py`：二维装置交互、状态机流程和教学风险评价。
- `app/pages/6_可燃极限计算.py`：Le Chatelier 规则分步解释。
- `app/pages/7_实验报告生成.py`：报告参数选择、预览和下载。

## 工具层

- `app_config.py`：软件名称、版本、安全声明、模块列表和风险颜色配置。
- `ui_theme.py`：全局页面样式和页脚。
- `ui_components.py`：页面标题、指标卡、说明卡、风险标签和步骤条。
- `chart_utils.py`：气体组成、风险仪表盘、场景对比和贡献图。
- `data_loader.py`：CSV 数据读取和文献样本筛选。
- `lfl_calculator.py`：LFL 常数读取、可燃比例、归一化和 LFL_mix。
- `risk_model.py`：空间浓度、R 值和风险等级。
- `report_generator.py`：Markdown / HTML 报告生成。

## 数据流

```text
CSV 数据 → data_loader → 页面筛选/选择 → lfl_calculator / risk_model → chart_utils → report_generator
```

虚拟实验页面会将最近一次实验记录保存到 `st.session_state["virtual_experiment_last"]`，报告页面可读取该记录作为默认参数。

## 异常处理

页面层对文件缺失、空数据、字段缺失和无法计算的情况提供提示。底层计算函数保持明确的异常行为，并由测试覆盖。

## 扩展原则

后续扩展应优先复用现有工具层函数。新增数据字段时，应同步更新数据字典、测试样例和页面字段解析逻辑。
