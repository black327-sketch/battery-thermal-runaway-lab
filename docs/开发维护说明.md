# 开发维护说明

## 开发环境

建议使用 Python 3.10 或更高版本。依赖安装：

```bash
pip install -r requirements.txt
```

## 本地运行

```bash
streamlit run app/main.py
```

## 测试

```bash
pytest tests/ -v
```

当前测试覆盖数据加载、LFL 计算和风险模型。修改核心数据解析或计算函数后必须运行测试。

## 代码维护原则

- 不修改核心计算函数的公式含义和风险阈值，除非同步更新测试和文档，并明确说明原因。
- 页面层负责交互、提示和展示；计算逻辑应放在 `app/utils/`。
- 新增图表优先放在 `chart_utils.py`。
- 新增通用 UI 片段优先放在 `ui_components.py`。
- 新增软件名称、版本、安全声明或模块配置时，优先修改 `app_config.py`。
- 新增数据字段时，同步更新 `docs/数据字典.md` 和相关页面解析逻辑。

## 数据维护流程

1. 备份或记录原 CSV 文件。
2. 按数据字典补充字段。
3. 确认数值单位，尤其是 LFL/UFL 的 `% vol`。
4. 检查 `source`、`DOI`、`reference`、`reliability_level` 等来源字段。
5. 使用 `app/utils/data_quality.py` 中的函数检查字段、空值、重复键和数值范围。
6. 运行 `pytest tests/ -v`。
7. 启动 Streamlit，检查文献数据库、虚拟实验、可燃极限计算和报告生成页面。

## 页面修改检查流程

1. 启动 `streamlit run app/main.py`。
2. 打开首页和 5 个子页面。
3. 检查 CSV 为空、筛选结果为空、字段缺失时是否有友好提示。
4. 检查文献数据库图表是否能正常渲染。
5. 检查虚拟实验是否能计算并保存最近一次记录。
6. 检查实验报告是否能在未先运行虚拟实验时手动生成。

## 新增功能流程

1. 先补充或更新相关文档，说明功能边界。
2. 再增加工具函数或数据结构。
3. 再增加页面入口或页面交互。
4. 最后增加或更新测试。
5. 运行测试并做页面冒烟检查。

## 禁止事项

- 不修改核心计算逻辑，除非有明确需求并同步修改测试和文档。
- 不修改风险等级阈值。
- 不添加真实危险实验步骤。
- 不编造文献来源、DOI、CAS、作者、单位或测试结果。

## 发布前检查

- README 中版本号与 `app/utils/app_config.py` 一致。
- `CHANGELOG.md` 已记录本次改动。
- 文档未写入未核实的 DOI、作者、单位、专利或软著编号。
- 页面安全边界表述清楚。
- 本地测试通过。
